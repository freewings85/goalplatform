# 业务线看板(按角色出内容 + 分页)— 设计

日期:2026-07-14

## 需求

- 进入某条业务线后,新增「📊 看板」菜单(所有登录用户可见;未进业务线时不显示)。
- **管理用户**看该业务线下**所有人**的目标和计划;**普通用户**只看**自己负责**的目标和计划。
- 数据量可能大,按"根条目"**分页**。
- 展示形式与工作台一致:目标 → 子目标 → 子子目标的树形甘特。

## 范围规则(服务端强制)

- 管理用户(`role=manager`):当前业务线 + 所选周期下的全部**顶层目标**(parent_id 为空),每个带完整子树。
- 普通用户:**自己负责(owner_user_id=me)的目标 + 完整子树**;若某目标的祖先也是自己负责的,只列祖先(不重复)。
- 未登录 → 401。

## 接口

`GET /api/board?business_line_id=&cycle_id=&page=`(新文件 `routers/board.py`,在 main.py 注册)

实现:一次查出业务线全部目标 → 内存建父子索引 → 按角色算根条目列表 → 按根分页(每页 5 个根,常量 `PAGE_SIZE`)→ 返回:

```json
{ "items": [<goal_dict,含 KR+阶段,与 /api/goals 同构>],
  "root_ids": [8, 12],
  "total": 12, "page": 1, "page_size": 5 }
```

- `items` 与现有 `/api/goals` 同构,前端拼树直接复用。
- `root_ids` 显式给根(普通用户的根不一定 parent_id 为空)。
- 周期过滤只作用在**根条目**上,子树完整展示。
- 翻页越界回落到最后一页;page 从 1 起。

## 前端(frontend/index.html)

- 侧边栏加 `data-nav="board"`「📊 看板」,仅 `state.lineId` 存在时显示。
- `board` 视图:面包屑「业务线 / <线名> · 看板」;周期切换(默认当前周期,复用工作台的);
  甘特渲染完全复用工作台现有函数(含交互);底部分页条(上一页 / 第 x / y 页 / 下一页)。
- 空态:普通用户无目标时显示"本周期没有你负责的目标"。
