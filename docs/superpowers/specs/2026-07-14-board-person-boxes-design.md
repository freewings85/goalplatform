# 看板按人分框 — 设计

日期:2026-07-14(改自 2026-07-14-role-board-design.md 的分页版)

## 需求

看板从"扁平分页甘特"改为**按人分框**:以顶层目标为粒度按负责人分组,一人一个固定大小的框体;框内滚动,框外整页滚动,**不再分页**。

## 规则

- 分组键 = 顶层目标的负责人;负责人未指定的树归「未指定负责人」框。
- 管理用户看所有人的框;普通用户只看自己参与的树所在的框(树级可见性不变)。
- 框排序与目标排序一致(树的 sort_order,谁的树靠前谁的框靠前)。

## 布局

- 一人一整行,纵向堆叠;框高固定(320px 常量):框头「👤 名字 · N 棵目标树」,框内纵向滚动,甘特时间轴横向框内滚动。
- 所有框共用同一时间范围(时间轴跨人对齐可比),行渲染与交互复用工作台现有函数。
- 周期切换保留(过滤树根);空态文案不变。

## 接口

`GET /api/board?business_line_id=&cycle_id=`:去掉 page/page_size,返回该用户可见的全部
`{items, root_ids}`。按人分组在前端做,后端不感知 UI 分组。

## 验证

三用户 API 对比(root_ids 与分页版一致,只是全量);前端 JS 语法检查 + 本地人工确认滚动;SQLite 回归。
