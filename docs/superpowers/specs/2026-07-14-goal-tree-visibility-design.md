# 目标树级可见性 — 设计

日期:2026-07-14

## 需求

业务线权限维持现状(所有登录用户可见可管理)。**目标**收紧:除管理用户外,普通用户只能看到**自己参与的目标树**。

## 规则(服务端强制,读接口)

- 管理用户(`role=manager`):所有目标树(现状不变)。
- 普通用户:**参与的树**——某顶层目标的子树(含顶层自身)里存在 `owner_user_id=我` 的目标,则**整棵树**可见,树内所有交互(编辑/加子目标/改阶段)不受限。
- 未登录调目标读接口 → 401。

## 改动

1. 新模块 `backend/scope.py`:`visible_tree_roots(goals, user)` — manager 返回全部顶层目标,normal 返回参与的树的顶层目标。goals.py / board.py 共用。
2. `GET /api/goals`:加 `require_user`;按业务线取全量建树 → 过滤可见树成员 → 按现有逻辑做周期过滤。
3. `GET /api/goals/{id}`:加 `require_user`;目标不在可见树内 → 403。
4. 看板 `GET /api/board`:普通用户的根改为参与树的顶层目标(替换"自己负责+去重");分页/周期过滤不变。
5. 前端:看板普通用户标签改「我参与的目标」;空态"本周期没有你参与的目标";其余零改动。

## 边界

- 普通用户没参与任何树:工作台/看板空态。
- 写接口(编辑/删除/审批)维持现状不加锁——UI 够不到不可见目标,与业务线权限口径一致。

## 验证

三用户(李明 manager、王芳只负责子目标 O1.1、陈磊负责 O2 树):王芳见 O1 整树含张伟的 O1.2、不见 O2;陈磊反之;李明全见;越权 get 403;未登录 401;看板分页复测;SQLite 回归。
